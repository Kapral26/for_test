1) После установки необходимо в gitbash
прописать адрес электронной почты, для этого:

git config --global user.name "Kapral ALeksandr"
git config --global user.email a.kapral26@gmail.com

2) Создать новый репозиторий

Перей в каталог
для win(используется не обратный слыщ как бычно, а обычный):
cd D:/....
и в нужном катологе используется комманда:
git init

3) Для добавления файла используется
в каталоге для добавления, используется комманда:
git add

если ввести:
git add . 

будет добавлены все файлы из каталога, если необходимо конкретный, то необходимо указать его имя

4) после изменения git их отследит и при запуске комманды:

git status

отобразится, что появились изменения в определееном файле

5) Если не добавить в git изменения и закоммитить, то в ветку попадет версия файла,
которая была добавлена ранее, поэтому ВАЖНО, чтобы перед комитом добавлять
в подготовленную версию фала

Пример:

*внес изменения*
git add *имя_файла*
*внес изменения*
git add *имя_файла*
git commit -m "Оставляешь комментарий к коммиту, где указываешь что было исправлено"

6)Все что делалось, до этого это была настройка на стороне локальной машины
для отправки коммитов на сервер, необходимо создать на сайт репозиторий,
в появившемся окне будут команды для привязки каталога и репозитория на сервере github
для привязки:
git remote add origin https://github.com/Kapral26/for_test.git

Если появилась ошибка fatal: remote origin already exists.:
Скорее всего, ссылка уже есть и ее необходимо почистить
для проверки:
git remote -v show
для очистки:
git remote rm origin
после чего вновь:
git remote add origin https://github.com/Kapral26/for_test.git

7)Для отправки в репозитрий:
git push -u origin master
